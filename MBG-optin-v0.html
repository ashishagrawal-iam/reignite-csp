OptIn V0 — announcement + opt-in

A focused working space to sharpen the कमाई कवच opt-in communication and add further screens. It starts with the two CleverTap full-pages carried over from v1 — the launch announcement and the mutual-oath opt-in — and will grow as we improve the flow.

Stage 1 · Announcement

The CSP hears about कमाई कवच (the MBG)

A full-page CleverTap takeover on Home — the biggest, clearest way to land the news within v1 limits.

Trigger: scheme launch / shortlistedCleverTap full-page
  • One-line promise up top (BV4): the CSP grasps it before reading anything else — "हर महीने ₹10,000 तक की कमाई पक्की" (the scheme is named कमाई कवच, never "बोनस").
  • The कमाई कवच promise is anchored up front — up to ₹10,000/month, paid at month-end, for the cold-start window (till September). No bonus clutter on this screen.
  • Single CTA "और जानें" opens the drilldown, Home-routed via the existing in-app listener.
Why full-page, not a banner here: awareness needs maximum signal once. After this, the banner + drilldown carry the ongoing story — so the full-page is reserved for moments that genuinely deserve a takeover.

Stage 2 · Opt-in

The CSP accepts कमाई कवच

A second full-page framed as a mutual oath — each side makes one vow — and captures acceptance via the CTA payload. This is the Mutual Compact, realised inside CleverTap.

Trigger: taps "और जानें" / launch follow-upCleverTap full-page (accept CTA)
  • A mutual oath, not a one-way offer: each side makes one promise — Wiom vows weekly leads, the CSP vows 60% installs. "यही आपकी कमाई पक्की करता है." This two-way framing is what makes the commitment stick.
  • The guarantee as safety net: "कम कनेक्शन मिलें तो भी हर महीने ₹10,000 तक पक्के" — so the CSP's promise carries no downside risk.
  • Accept CTA "मैं वादा करता हूँ" — the button payload is captured by the existing CleverTap listener and recorded as opt-in (no new screen needed).
  • Plain terms below: ≥3 leads/month required, कमाई कवच paid at month-end.
Why the oath framing: a recorded mutual vow turns a broadcast into a relationship the CSP feels accountable to — the persistence the Cold Start floor demands. "मैं वादा करता हूँ" lands far stronger than "मैं तैयार हूँ".

Stage 3 · After the oath

Audit gate + quiz → कमाई कवच चालू

The moment the CSP taps "मैं वादा करता हूँ", an invisible router checks the नेटबॉक्स माफ़ी योजना audit and branches. Case 1 (audit pending) sends them to the audit (external browser) and handles the return. Case 2 (audit done) takes them through a one-question comprehension quiz. Both converge on कमाई कवच being switched on.

मैं वादा करता हूँ→ऑडिट हुआ?
Case 1 · नहींऑडिट पूरा करें→URL (browser)→वापसी पर दोबारा जाँच→क्विज़
Case 2 · हाँक्विज़→सही जवाब→कमाई कवच चालू 🎉
Trigger: taps "मैं वादा करता हूँ"CleverTap full-pages + external URL
  • Router re-checks on app-resume: CleverTap in-app doesn't fire on return from the browser, so the app re-fetches audit status on foreground — and routes to done → quiz, processing → "जाँच में है", or still pending → gentle retry (never "you didn't finish it").
  • Quiz is comprehension, not an exam: a wrong tap opens a warm teach screen and loops back — unlimited retries, no red, no "गलत", no lockout.
  • Real eligibility = oath AND audit-done AND quiz-passed, persisted server-side. Month-end payout keys off this flag, not the oath tap.
  • Fast-path: if the audit is N/A or already cleared, Case 1 is skipped — straight to the quiz.
  • "बाद में करें" anywhere → Home with progress saved; the oath is never re-asked, and a soft home banner re-invites.
  • Offline-safe: status fetch / eligibility write failures show a non-blaming "थोड़ी देर में फिर कोशिश करें", and the quiz pass is cached server-side.
Naming locked: the success state is कमाई कवच — never "सुरक्षा कवच" (a one-off slip; the CSP just committed to कमाई कवच end-to-end) and never "बोनस" (that's the separate 16th quality bonus). Params locked to the CSP model: up to ₹10,000/month · gate 60% · ≥3 connections · paid month-end · till September.
Heads-up for the team: internal files (MBG - internal file, comms-journey) still carry the old economics (70% / ₹5,000). These CSP screens use the current 60% / ₹10,000. Lock one source of truth before build.

Case 2 / convergence · Quiz → eligible

Reached when the audit is done (first check, on return, or N/A). Four comprehension questions gate entry; tapping an option is the action. Each correct answer advances; a wrong one opens that question’s teach screen (with the calculation) and loops back. All four correct switches on कमाई कवच. (Correct: B, C, B, D.) On tap the chosen option flashes green ✓ if correct (then advances) or red ✕ if wrong — revealing the correct option — before opening the teach screen. (Live in the Prototype tab.)
